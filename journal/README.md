# Project Journal

Tracks the evolution of this project — not just what changed, but how the thinking developed, what inspired each move, and what was learned.

## Files

- **timeline.md** — Chronological log (short entries, one per milestone)
- **entries/** — Deeper write-ups for significant thinking shifts
- **origins.md** — How this idea was born and the thinking that shaped it

## When to Write

**Timeline entry** when:
- A design decision is made or changed
- New research reshapes the architecture
- A prototype produces surprising results
- The core thesis evolves

**Full journal entry** when:
- A thinking shift occurred (the "aha" moment matters)
- An external paper/system changed the design direction
- An implementation attempt revealed something unexpected
- The relationship to The Companion project changes

## Entry Format

```markdown
---
date: YYYY-MM-DD
type: insight | decision | research | prototype | pivot
---

# [Title]

## What Shifted
[The change in thinking or design]

## Why
[What triggered it — conversation, research, implementation]

## Implications
[How this changes the architecture or direction]
```
